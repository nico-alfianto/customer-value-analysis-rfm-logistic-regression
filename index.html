Project Overview

Not all customers are created equal. This project analyzes Online Retail transaction data using RFM Analysis and Logistic Regression to answer two critical business questions:

  • Which customers drive the most revenue?
  • Can we predict high-value customers before they spend big?

Key Finding: Pareto Principle

High-value customers contribute 84.9% of total revenue, while only making up 33.3% of the customer base. Monetary value is the strongest predictor, followed by Frequency and Recency.

Methodology

  1. Data Cleaning: Handle returns, missing CustomerID, outliers
  2. RFM Analysis: Calculate Recency, Frequency, Monetary scores
  3. Segmentation: Group into High, Mid, Low value tiers
  4. Logistic Regression: Predict probability of being high-value
  5. Business Action: Retention, Upsell, Reactivation strategies

RFM Calculation

snapshot_date = df['InvoiceDate'].max() + timedelta(days=1)

rfm = df.groupby('CustomerID').agg({
  'InvoiceDate': lambda x: (snapshot_date - x.max()).days,
  'InvoiceNo': 'nunique',
  'Revenue': 'sum'
})
rfm.columns = ['Recency', 'Frequency', 'Monetary']

SQL Logic Example

-- Calculate total revenue per customer
SELECT CustomerID, SUM(Quantity * UnitPrice) AS Revenue
FROM online_retail
GROUP BY CustomerID;

-- Segment customers using NTILE
SELECT *, NTILE(3) OVER (ORDER BY Revenue DESC) AS revenue_segment
FROM ( ... ) t;
